🔐 Bloom CC — Password Migration Tool

One-time tool to convert plain-text passwords in /users to bcrypt hashes. After running this, the plain password field is removed and replaced with password_hash. The updated cc-admin.html handles both formats during transition.
⚠️ Run this only ONCE per project. After all users are migrated, plain-text passwords are gone from the database. If you re-run the seeder afterwards (which writes hashes now), this tool isn't needed again.
How does bcrypt work? (click to expand)
bcrypt is a one-way hashing function — it converts a password like "3664" into something like $2a$10$N9qo8uLOickgx2ZMRZ.... The hash CANNOT be reversed back to the original password. Login works by hashing the typed password the same way and comparing the two hashes. Even if someone dumps the entire users collection, they cannot extract the original passwords. The cost factor (10 here) makes brute-force attacks prohibitively slow.
